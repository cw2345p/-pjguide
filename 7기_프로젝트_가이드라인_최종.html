이 프로젝트에서 가장 중요한 한 가지
"작동하는 작은 것 하나"가 "완성 못 한 거대한 계획"보다 훨씬 가치 있습니다. MVP를 먼저 만들고, 기능은 나중에 붙입니다.
01
설계 먼저, 코드 나중

코드를 쓰기 전에 데이터가 어디서 들어오고, 어디로 가고, 어떻게 나오는지 그림으로 그릴 수 있어야 합니다. 그림을 못 그리면 코드를 시작하지 않습니다.

02
AI는 보조, 내가 주도

AI가 만든 코드를 이해 없이 복붙하지 않습니다. AI에게 묻기 전에 "내가 어떻게 풀 것인가" 를 30초 먼저 생각합니다. 도구가 안 되면 더 단순한 방법을 씁니다.

03
막히면 2시간 안에 보고

혼자 하루 종일 고민하지 않습니다. 2시간 이상 막히면 팀 채널에 "어디서 막혔고 어떻게 시도했는지" 메시지를 보냅니다.

서비스 설계 — 데이터 흐름을 그림으로 그린다
목표: 데이터 흐름도 완성 (코드 없이 그림으로)
먼저 이 3가지 질문에 답하세요 (AI 없이, 팀이 직접)

Q2. 그 데이터가 어떻게 처리되나요?

예) CV모듈 입력: mp4 파일 / 출력: {gaze_score: 75, gesture_count: 12}

4 AI 올바른 사용 기준
✅ 이렇게 쓰세요
  • 에러 메시지를 붙여넣고 원인 물어보기
  • "이 코드가 왜 작동 안 하냐" 질문
  • 개념 이해가 안 될 때 설명 요청
  • 내가 쓴 코드 리뷰 요청
  • 짧은 유틸 함수 작성 요청
  • API 응답 파싱 방법 물어보기
❌ 이렇게 쓰지 마세요
  • "발표코치 앱 전체 만들어줘"
  • AI 코드를 이해 없이 복붙
  • 도구가 안 되면 손 놓기
  • 팀 상의 없이 AI 답변으로 방향 바꾸기
  • AI가 짠 코드를 내 코드처럼 제출
  • 안티그래비티 등 도구 의존 → 안 되면 멈춤
⚠️ 도구(안티그래비티 등)가 안 될 때 → 도구 없이 다른 AI (Chat GPT, Gemini 등으로) 직접 구현하는 방법으로 전환합니다. 도구가 안 된다고 프로젝트가 멈추면 안 됩니다.

6 위험신호 & 대처법
